mlist: Verwaltung von Mailling-Listen

Die Verwaltung von Mailling-Listen erfolgt in einem einfachen Dateiformat:

"Teilneher Eins" <teilnehmer.eins@provider.de>
"Teilneher Zwei" <teilnehmer_zwei@provider.com>

Diese Datei kann entweder auf dem eigenen Rechner oder auf dem Server verwaltet werden.

  • Bei der Verwaltung auf dem eigenen Rechner muss die Datei nach einer Änderung auf den Server kopiert werden. Ein geeignetes Programm dafür ist WinSCP (Windows) oder gftp (Linux).
  • Soll die Datei direkt auf dem Server bearbeitet werden, meldet sich der Bearbeiter am Server an (Windows: Putty, Linux: Shell), editiert die Datei (Befehl: nano <liste>.mlist) und speichert sie.

Nach einer Änderung muss diese aktiviert werden. Dazu muss sich der Bearbeiter auf dem Server einloggen (Windows: Putty, Linux: Shell) und auf dem Server das Kommando update-mlist absetzen.
